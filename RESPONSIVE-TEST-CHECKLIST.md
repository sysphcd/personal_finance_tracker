# 響應式設計測試檢查清單

## 需求 3.5: 響應式設計驗證

本檢查清單用於驗證個人記帳本應用程式的響應式設計是否符合需求。

### 測試環境準備

1. **桌面瀏覽器測試**
   - [ ] Chrome/Edge (開發者工具 - 裝置模擬器)
   - [ ] Firefox (響應式設計模式)
   - [ ] Safari (如果使用 macOS)

2. **實際裝置測試** (建議)
   - [ ] 手機 (iOS 或 Android)
   - [ ] 平板電腦

### 測試檔案

- `index.html` - 主應用程式
- `test-responsive.html` - 視覺化響應式測試
- `run-responsive-tests.html` - 自動化測試頁面

---

## 測試項目

### 1. 基本響應式設定 ✓

#### 1.1 Viewport Meta 標籤
- [ ] HTML 包含 `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- [ ] 在行動裝置上頁面不會被縮小

**驗證方法**: 檢查 `index.html` 的 `<head>` 區段

---

### 2. 媒體查詢實作 ✓

#### 2.1 中等螢幕 (≤768px)
- [ ] CSS 包含 `@media (max-width: 768px)` 媒體查詢
- [ ] 容器邊距調整為 20px
- [ ] 輸入區域改為垂直排列 (flex-direction: column)
- [ ] 所有輸入欄位和按鈕寬度為 100%

**驗證方法**: 
1. 開啟 `run-responsive-tests.html`
2. 調整瀏覽器視窗寬度至 768px 以下
3. 觀察輸入區域是否垂直排列

#### 2.2 小螢幕 (≤480px)
- [ ] CSS 包含 `@media (max-width: 480px)` 媒體查詢
- [ ] 容器邊距進一步縮小至 10px
- [ ] 標題字體大小調整為 1.3em
- [ ] 支出項目改為垂直排列
- [ ] 刪除按鈕寬度為 100%

**驗證方法**:
1. 調整瀏覽器視窗寬度至 480px 以下
2. 觀察支出項目佈局變化

---

### 3. 觸控目標尺寸 ✓

#### 3.1 最小觸控尺寸 (44x44px)
- [ ] 新增按鈕高度 ≥ 44px
- [ ] 金額輸入欄位高度 ≥ 44px
- [ ] 類別選單高度 ≥ 44px
- [ ] 刪除按鈕高度 ≥ 36px (可接受較小尺寸)

**驗證方法**:
1. 開啟 `run-responsive-tests.html`
2. 調整視窗至 768px 以下
3. 檢查控制台輸出的觸控目標尺寸測試結果
4. 或使用瀏覽器開發者工具檢查元素的計算高度

---

### 4. 字體大小調整 ✓

#### 4.1 防止 iOS 自動縮放
- [ ] 輸入欄位字體大小 ≥ 16px (在 ≤768px 時)
- [ ] 選單字體大小 ≥ 16px (在 ≤768px 時)

**驗證方法**:
1. 在 iOS 裝置上測試
2. 點擊輸入欄位時頁面不應自動縮放
3. 或在開發者工具中檢查計算的字體大小

#### 4.2 可讀性
- [ ] 標題在小螢幕上調整為 1.5em (768px) 或 1.3em (480px)
- [ ] 總計顯示在小螢幕上調整為 1.1em (768px) 或 1em (480px)
- [ ] 支出項目文字在小螢幕上調整為 0.95em

---

### 5. 佈局調整 ✓

#### 5.1 容器調整
- [ ] 桌面: 最大寬度 600px，置中顯示
- [ ] 平板 (≤768px): 邊距 20px
- [ ] 手機 (≤480px): 邊距 10px

#### 5.2 輸入區域
- [ ] 桌面: 水平排列 (flex-direction: row)
- [ ] 行動裝置 (≤768px): 垂直排列 (flex-direction: column)
- [ ] 間距適當調整

#### 5.3 支出清單項目
- [ ] 桌面/平板: 水平排列 (文字和按鈕並排)
- [ ] 小手機 (≤480px): 垂直排列 (按鈕在下方，全寬)

**驗證方法**:
1. 開啟 `test-responsive.html`
2. 調整視窗寬度觀察佈局變化
3. 測試寬度: 1024px, 768px, 480px, 320px

---

### 6. 易用性測試 ✓

#### 6.1 觸控操作
- [ ] 所有按鈕易於點擊，不會誤觸
- [ ] 輸入欄位易於聚焦
- [ ] 下拉選單易於操作

#### 6.2 視覺清晰度
- [ ] 文字清晰可讀
- [ ] 元素間距適當，不擁擠
- [ ] 顏色對比度足夠

#### 6.3 功能完整性
- [ ] 所有功能在小螢幕上正常運作
- [ ] 沒有內容被裁切或隱藏
- [ ] 捲動流暢

**驗證方法**:
1. 在實際行動裝置上測試
2. 嘗試新增和刪除支出記錄
3. 檢查所有互動是否順暢

---

## 測試步驟

### 自動化測試
1. 開啟 `run-responsive-tests.html`
2. 觀察控制台輸出
3. 調整視窗寬度至不同尺寸
4. 確認所有測試通過

### 手動測試
1. 開啟 `index.html` 或 `test-responsive.html`
2. 使用瀏覽器開發者工具的裝置模擬器
3. 測試以下裝置預設:
   - iPhone SE (375x667)
   - iPhone 12 Pro (390x844)
   - iPad (768x1024)
   - iPad Pro (1024x1366)
4. 測試橫向和直向模式

### 實際裝置測試
1. 在手機瀏覽器中開啟應用程式
2. 測試所有互動功能
3. 檢查觸控體驗
4. 驗證字體大小和可讀性

---

## 測試結果

- [ ] 所有功能運作正常

### 平板 (≤768px)

- [ ] 觸控目標尺寸適當
- [ ] 佈局適應良好

- [ ] 支出項目垂直排列

- [ ] 所有內容可見且易於操作
